- 顶层 package.json 同时管理前端 (Vue/Vite/Vitest) 与 Tauri CLI 脚本 (`tauri dev`/`tauri build`)，是前后端统一的构建入口。
- `vite.config.ts` 通过环境变量 `VITE_PLATFORM` 决定 `@adapter` 别名指向 `src/adapters/tauri-adapter` 还是 `src/adapters/web-adapter`，并在 web 模式下将 `@tauri-apps/api` 外部化，使同一份业务代码可同时在浏览器与 Tauri 窗口运行。
- 前端通过 `@tauri-apps/api` 调用 Rust 侧注册的命令；Rust crate 位于 `src-tauri/`，使用 `tauri = "2"`、`tokio`、`zip`/`flate2` 等依赖实现文件与解压等原生能力，形成「前端适配器层 → Tauri IPC → Rust 原生」三层边界。
- 测试由 Vitest 驱动，配合 jsdom 对纯前端逻辑进行单元测试，Tauri 相关命令则通过 `tauri:dev`/`tauri:build` 走完整打包链路。
