模块由三个 Rust 源文件按职责分层组织：
- `commands.rs` 作为 Tauri 命令入口，每个函数通过 `#[tauri::command]` 注册到 IPC 总线；所有接受外部路径的参数先经内部 `validate_path`（逐段拒绝 `..` + 可选 `allowed_base` 前缀校验）再委派给底层实现，并通过 `app_data_base` 将写入范围限制在应用数据目录。
- `file_ops.rs` 提供纯函数式 IO 能力：基于 `memmap2::Mmap` 的 `mmap_read`（含 offset+length 溢出检查与越界校验）和递归 `list_files`（返回 `FileMeta`），不依赖 Tauri，可被单元测试直接调用。
- `decompress.rs` 提供 `decompress_zip`（`zip` crate）与 `decompress_gzip`（`flate2`），对 zip 条目名做 `sanitize_zip_entry_name` 净化并执行 Zip Slip 防护（`canonicalize` + `starts_with`），同时以 `MAX_TOTAL_SIZE=1GB` 累计大小防止 zip bomb，并以 `DecompressResult { success, files, error }` 三态聚合结果而非抛出异常。
依赖方向单向：`commands` → `file_ops` / `decompress`，两者均向上抛 `crate::error::AppError`，保持命令层薄且无业务逻辑。阻塞型 I/O（mmap、walk_dir）通过 `tokio::task::spawn_blocking` 迁出异步运行时以避免阻塞事件循环。
