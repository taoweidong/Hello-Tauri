- 根 `package.json` 同时声明业务依赖与 `@tauri-apps/cli`，并通过 `tauri:dev`/`tauri:build` 脚本把构建入口指向 `src-tauri/` Rust crate，形成「单仓库、双产物」的统一入口。
- `vite.config.ts` 基于环境变量 `VITE_PLATFORM`（默认 `web`）将 `@adapter` 别名动态指向 `src/adapters/tauri-adapter` 或 `src/adapters/web-adapter`；web 模式下通过 rolldown `external` 把 `@tauri-apps/api` 外部化，使业务层仅依赖抽象适配器接口。
- 运行时边界：Vue 组件 → `@adapter` 注入的适配实现 → `@tauri-apps/api` invoke → `src-tauri/` 中注册的 Rust 命令 → 操作系统文件/解压等原生能力，构成「前端适配器层 → Tauri IPC → Rust 原生」三层隔离。
- 测试由 Vitest 驱动，配合 jsdom 对纯前端逻辑进行单元测试；涉及 IPC 的完整链路则通过 `tauri dev` 走真实 Tauri 窗口验证。
