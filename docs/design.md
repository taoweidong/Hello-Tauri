# 跨平台日志解析工具完整设计方案

## 1. 项目概述与技术选型

### 1.1 项目目标
构建一个支持 **Web 网站** 与 **Windows 单文件 EXE** 双端运行的日志解析工具。核心能力包括：多压缩包上传、递归解压、独立文件树展示、文件预览及公共信息汇总。系统需具备极高的扩展性，新增压缩格式或文件格式仅需添加插件，无需修改核心代码。

### 1.2 技术栈约束
-   **前端框架**：Vue 3 + TypeScript + Vite
-   **桌面框架**：Tauri (Rust)
-   **架构模式**：微内核 + 插件化
-   **设计原则**：非必要不引入新语言/技术，核心逻辑尽量 TS 化，Rust 仅作为系统级桥接

### 界面各区域

#### 4.2.1 顶部全局公共信息栏
-   **聚合统计**：实时显示所有已上传压缩包的汇总信息（总包数、总原始大小、总解压后大小、总文件数、已解压文件数、整体耗时）
-   **全局搜索**：输入关键词后，在所有已解压文件中并行搜索，结果以列表形式展示，点击跳转至对应文件的对应行
-   **批量操作**：一键清空、全部导出、批量重新解压

#### 4.2.2 左侧压缩包列表与独立文件树
-   **多包并列展示**：每个上传的压缩包以独立卡片形式纵向排列，互不干扰
-   **独立文件树**：每个压缩包卡片内嵌完整的文件树组件，支持独立折叠/展开、独立搜索过滤
-   **状态可视化**：
    -   🟢 已完成：绿色对勾
    -   🔵 解压中：蓝色旋转动画 + 百分比进度条
    -   🟡 排队中：灰色时钟图标
    -   🔴 失败：红色感叹号 + 错误摘要
-   **交互**：点击文件节点 → 中间工作区新开标签页或激活已有标签页；右键菜单支持预览/导出/复制路径/查看元数据

#### 4.2.3 中间预览工作区
-   **多标签页管理**：支持拖拽排序、关闭、固定、拆分视图（左右/上下分屏对比）
-   **动态渲染器**：根据文件类型从插件注册中心获取对应 Vue 组件，通过 `<component :is>` 动态挂载
-   **预览工具栏**：
    -   文本类：字号调节、换行开关、行号开关、编码切换
    -   CSV类：分隔符设置、表头固定、列排序/筛选
    -   通用：关键字高亮正则配置、搜索结果导航
-   **性能状态栏**：底部常驻显示 `当前行/总行数 | 可视区内存占用 | 加载耗时 | 所用解析插件`

#### 4.2.4 右侧属性面板
-   **上下文感知**：选中的是压缩包 → 显示包级元数据；选中的是文件 → 显示文件级元数据
-   **插件配置注入**：当前文件的解析插件可通过 `getConfigSchema()` 声明配置项，属性面板自动渲染对应的表单控件
-   **关联信息**：显示文件在压缩包内的完整路径、嵌套层级、父压缩包链路

## 5. 扩展性保障机制

### 5.1 新增压缩格式流程
1.  新建 TS 模块，实现 `ICompressionPlugin` 接口
2.  若需原生解压能力，编写对应 WASM 模块或 Rust 函数
3.  在插件清单配置文件中注册
4.  **核心代码零修改**

### 5.2 新增文件格式预览流程
1.  新建 Vue 组件作为渲染器
2.  新建 TS 模块，实现 `IFileParserPlugin` 接口，关联上述组件
3.  可选：声明配置 Schema，属性面板自动生成配置表单
4.  在插件清单配置文件中注册
5.  **核心代码零修改**

### 5.3 插件隔离与安全
-   每个插件运行在独立作用域，异常不影响核心引擎
-   插件通过白名单 API 访问系统资源，禁止任意文件读写
-   支持运行时启用/禁用插件，热更新无需重启

## 6. 构建与部署策略

| 目标平台 | 构建方式 | 产物 | 备注 |
| :--- | :--- | :--- | :--- |
| **Web** | `vite build --mode web` | 静态 HTML/JS/CSS + WASM | 部署至 Nginx/OSS |
| **Windows EXE** | `tauri build` | 单文件 `.exe` (~8-15MB) | Rust 后端 + WebView2 |
| **开发调试** | `vite dev` + `tauri dev` | HMR + Rust 热重载 | 共享同一套源码 |

> **关键备注**：通过 Vite 环境变量 `VITE_PLATFORM=web|tauri` 在编译期条件导入对应 PlatformAdapter，确保 Web 产物不含 Tauri 依赖，EXE 产物按需包含 WASM 回退方案。
