# 压缩文件说明
上传的压缩文件格式固定，解压之后的文件类型固定，存在缺失或者增多的场景，如果有新增的文件，则提示不支持解压展示

示例1：
a1.zip
    - APPLOG1
    - APPLOG2
    - APPLOG3
    - MSGLOG1
    - MSGLOG2
    - MSGLOG3
    - sample.json
    - sample.csv
    - sample.txt
    - VERSION.txt
    - sample.log


示例2：
a2.zip
    - APPLOG1
    - APPLOG2
    - MSGLOG1
    - MSGLOG2
    - MSGLOG3
    - sample.json
    - sample.txt
    - VERSION.txt
    - sample.log
    - sample_table_tree.csv



# 文件说明

- VERSION.txt：版本说明文件，每个压缩文件中必须有，如果这个文件不存在，说明压缩文件不支持，不是目标压缩文件，当前工具不支持
- sample.json： json类型的日志，非必须文件，按照存在压缩文件中，按照json格式展示
- sample.txt： 文本类型的日志，非必须文件，按照存在压缩文件中，直接文本展示
- sample.log： 日志类型的日志，非必须文件，按照存在压缩文件中，按照日志拆分成表格展示
- sample.csv   csv类型日志，非必须文件，表格展示
- sample_table_tree.csv csv类型日志，表格数据，每一行点击时，加载数据中的树形结构数据，注意布局左右布局，左侧表格，右侧树形数据




# 严格遵守

- 要求一种类型的文件，对应一个解析数据的TS，和一个展示数据vue的页面
- 后续会新增其他类型文件，支持快速扩展
